**FakeAPI** — сервис для разработки и тестирования, который позволяет запускать Mock API-проекты без написания серверного кода.

[Сайт](https://fake-api.ru/) [Telegram](https://t.me/fake_api_ru)

Сервис полезен, когда нужно:
- начать фронтенд-разработку до готовности реального API;
- тестировать интеграции и edge-cases;
- быстро собрать прототип для демо или пилота;
- стабилизировать автотесты за счёт предсказуемых ответов API.

## Как это работает

Каждый проект в FakeAPI получает собственный поддомен. Это позволяет изолировать окружения по командам, фичам или этапам разработки.

Проект может быть:
- **публичным** — доступен без дополнительной авторизации;
- **защищённым** — с доступом по токену.

## Что можно настроить

В проекте можно гибко описывать поведение API:
- создавать эндпоинты с нужными HTTP-методами;
- задавать несколько вариантов ответов (например, `200`, `400`, `401`, `500`);
- эмулировать ошибки, задержки и нестандартные сценарии;
- менять структуру ответа под разные стадии разработки.

## Для кого подходит FakeAPI

FakeAPI особенно полезен:
- фронтенд-разработчикам;
- QA-инженерам и авто-тестировщикам;
- продакт-командам для быстрых демонстраций;
- интеграционным командам, которым важно тестировать контракт API заранее.

## Почему это удобно

- Быстрый старт: не нужен отдельный сервер и рутинная настройка бэкенда.
- Гибкость: легко управлять сценариями ответов под конкретные тесты.
- Безопасность: для приватных проектов можно ограничивать доступ токеном.
- Экономия времени: команда параллелит работу без блокировок между фронтом и бэком.
