Это руководство поможет вам за несколько минут создать полностью рабочий mock API: от регистрации проекта до первого реального HTTP-запроса.

---

## 1. Создайте проект

Проект — это изолированное пространство для ваших mock-эндпоинтов. Каждый проект получает собственный **поддомен**, через который и будут идти все запросы.

Введите название проекта и нажмите «Сохранить». После создания перейдите в проект и вы увидите поддомен вида:

![image](https://fake-api.ru/files/articles/019d1915-6377-7160-a3db-e447a734fb79/263afcfc-e1fa-4889-9053-47242c3e9329_Снимок_экрана_2026-03-23_222756.png)

Это и есть базовый URL вашего mock API.

> По умолчанию проект публичный. При необходимости можно настроить два независимых уровня защиты:
>
> - **Токен проекта** — защищает доступ ко всему проекту. Создаётся в «Настройки доступа» → «Проект», передаётся в заголовке `X-Project-Token`.
> - **Аутентификация эндпоинта** — имитирует защищённый API на уровне конкретного эндпоинта. Создаётся в «Настройки доступа» → «Аутентификация» (поддерживается ApiKey, JWT, Basic), затем подключается к нужному эндпоинту при его создании или редактировании.

---

## 2. Создайте эндпоинт

Эндпоинт описывает HTTP-запрос, который будет обрабатывать ваш mock.

Укажите:

- **Метод** — `GET`, `POST`, `PUT`, `PATCH`, `DELETE` и др.
- **Путь** — например, `/users` или `/users/{id}`

Путь может содержать параметры в фигурных скобках: `/orders/{id}`. Они участвуют в сопоставлении входящего запроса с нужным ответом.

При желании эндпоинты можно объединять в **группы** — для удобной навигации, когда их становится много.

---

## 3. Добавьте ответ

Ответ — это то, что клиент получит при обращении к эндпоинту.

Укажите:

- **HTTP-статус** — например, `200`, `201`, `404`
- **Тип** — JSON или текст
- **Тело ответа** — объект, массив или строка

Пример тела для `GET /users`:

```json
[
  { "id": 1, "name": "Alice" },
  { "id": 2, "name": "Bob" }
]
```

Одному эндпоинту можно добавить несколько ответов — для разных сценариев (успех, ошибка валидации, 404 и т.д.). Нужный ответ выбирается по match-критериям: параметрам пути, заголовкам, cookies или телу запроса.

---

## 4. Сделайте запрос

Всё готово. Обращайтесь к эндпоинту через поддомен проекта:

```bash
curl https://a1b2c3d4e5f6789012345678901234ab.fake-api.ru/users
```

Вы получите тело ответа, которое указали на предыдущем шаге.

---

Это минимально необходимый набор шагов. Дальше можно углубляться: настраивать схемы валидации, задержки, аутентификацию и сложную логику сопоставления запросов.
